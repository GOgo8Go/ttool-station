# i18n 翻译完整性检查工具

此工具用于自动检查项目中所有语言翻译的完整性，确保没有缺失或空的翻译项。

## 功能

- 检查所有语言文件是否包含基准语言（中文）的所有翻译键
- 检测空翻译项
- 生成翻译完成度报告
- 检测潜在的未翻译内容

## 使用方法

### 通过 npm 命令运行

```bash
npm run check-i18n
```

### 直接运行脚本

```bash
npx tsx scripts/check-i18n-completeness.ts
```

## 报告内容

运行检查工具后，将生成以下信息：

1. **翻译状态**: 每种语言的翻译覆盖率
2. **缺失键**: 某种语言中缺失的翻译键
3. **空翻译**: 所有语言中的空翻译项
4. **潜在问题**: 可能未翻译的值
5. **完成度总结**: 每种语言的翻译完成度百分比

## 输出示例

```
🔍 i18n 翻译完整性报告

基准语言: zh (共 1165 个翻译键)

📋 en 语言翻译状态:
 总键数: 1169/1165 (100.34% 覆盖率)
  ✅ 所有键都已翻译
  🚫 空翻译 (15 个):
    - tool.paint.name
    - tool.paint.desc
    ...

📊 翻译完成度总结:
  en: 1154/1169 非空翻译 (98.72% 完成度)
  fr: 1165/1165 非空翻译 (100.00% 完成度)
 ...
```

## 检查逻辑

- 使用中文 (`zh`) 作为基准语言
- 检查其他语言是否包含中文中的所有翻译键
- 检查所有语言中的空字符串翻译
- 检测可能未翻译的技术术语（如 "URL / URI", "HSV / HSB" 等）

## 集成到 CI/CD

可以将此检查集成到持续集成流程中，确保翻译完整性：

```bash
# 在构建流程中添加检查
npm run check-i18n
```

如果需要，可以根据返回码来判断是否继续构建流程。
